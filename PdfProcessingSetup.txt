## Simulation Mode

The system currently operates in simulation mode which demonstrates the complete workflow using sample questions. This allows you to test all functionality without requiring additional libraries.

## Production Setup (Optional Enhancement)

To enable processing of actual PDF files instead of simulation, you can install the following Java libraries:

### Required Libraries

1. **Apache PDFBox** - For PDF text extraction
   - Download from: https://pdfbox.apache.org/download.cgi
   - Required JARs:
     - pdfbox-2.x.x.jar
     - fontbox-2.x.x.jar
     - commons-logging-1.x.x.jar

2. **Apache Commons FileUpload** - For handling file uploads
   - Download from: https://commons.apache.org/proper/commons-fileupload/
   - Required JARs:
     - commons-fileupload-1.x.x.jar
     - commons-io-2.x.x.jar

### Installation Steps

1. Download the required JAR files from the official sources
2. Place the JAR files in the `web/WEB-INF/lib/` directory of your project
3. Restart your web server (Tomcat/Jetty) to load the new libraries
4. The PDF upload functionality will automatically switch to process actual PDF files

### Maven Dependencies (Alternative)

If your project uses Maven, add these dependencies to your pom.xml:

```xml
<dependency>
    <groupId>org.apache.pdfbox</groupId>
    <artifactId>pdfbox</artifactId>
    <version>2.0.29</version>
</dependency>
<dependency>
    <groupId>commons-fileupload</groupId>
    <artifactId>commons-fileupload</artifactId>
    <version>1.5</version>
</dependency>
```
